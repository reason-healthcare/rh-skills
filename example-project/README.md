# Example Project

This directory is a reference implementation of a **target repo** using the HI Skills Framework. It shows how a clinical team would structure their repository.

## `topics/diabetes-screening/`

A complete example clinical topic for diabetes screening, showing:

- `l1/` — raw discovery artifacts (guidelines, research notes)
- `l2/` — semi-structured YAML artifacts derived from L1
- `l3/` — computable YAML artifacts (FHIR-compatible)
- `plans/` — plan artifacts generated by framework agent skills
- `tracking.yaml` — workflow audit log with artifact checksums
- `TOPIC.md` — topic description

## Target Repo Structure

A clinical team using this framework would create their own repo with this layout:

```
my-clinical-repo/
├── topics/
│   ├── diabetes-screening/
│   ├── sepsis-detection/
│   └── hypertension-management/
└── .agents/
    └── skills/
        ├── hi-discovery/SKILL.md
        ├── hi-ingest/SKILL.md
        ├── hi-extract/SKILL.md
        ├── hi-formalize/SKILL.md
        ├── hi-verify/SKILL.md
        └── hi-status/SKILL.md
```

The `hi` CLI commands ship with this framework repo. Agent skills are distributed from `skills/.curated/` and installed into target repos at `.agents/skills/`.
