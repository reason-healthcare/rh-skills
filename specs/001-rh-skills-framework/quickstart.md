## Prerequisites

Install required CLI dependencies:

```bash
# macOS
brew install yq jq

# Ubuntu/Debian
apt-get install yq jq curl
```

Install the `rh-skills` CLI:

```bash
# Clone the repository
git clone https://github.com/your-org/rh-skills
cd rh-skills

# Add bin/ to your PATH
export PATH="$PATH:$(pwd)/bin"

# Verify
hi --version
```

For local LLM testing (recommended for development), install Ollama:

```bash
# macOS
brew install ollama
ollama serve &         # Start in background
ollama pull mistral    # Download a model

export LLM_PROVIDER=ollama
export OLLAMA_MODEL=mistral
```

## Authoring Your First Skill

### Step 1: Scaffold the skill

```bash
rh-skills init diabetes-screening \
  --description "Evidence-based screening criteria for Type 2 diabetes" \
  --author "Jane Smith"
```

This creates:
```
skills/diabetes-screening/
├── SKILL.md            ← Edit this: the agent prompt for your skill
├── tracking.yaml       ← Do not edit manually
├── l1/
│   └── discovery.md    ← Start here: paste raw notes, references, transcripts
├── l2/                 ← Generated by rh-skills promote
├── l3/                 ← Generated by rh-skills promote
└── fixtures/           ← Add test cases here
```

### Step 2: Populate L1 artifacts

Open `skills/diabetes-screening/l1/discovery.md` and add your raw discovery material — clinical guidelines text, interview notes, reference excerpts. No structure required.

You can add additional L1 files freely:
```bash
cp ada-guidelines.txt skills/diabetes-screening/l1/ada-guidelines-excerpt.md
```

### Step 3: Derive L2 artifacts (L1 → L2)

When your L1 content is ready, derive semi-structured L2 artifacts. One L1 can produce multiple L2 artifacts:

```bash
# Derive 2 L2 artifacts from one L1 artifact
rh-skills promote diabetes-screening ada-guidelines-excerpt --to l2 --count 2
```

This scaffolds:
```
l2/
├── concept-1.yaml    ← Pre-populated template; fill in the fields
└── concept-2.yaml
```

Open each generated YAML and fill in the domain content. Required fields are marked; optional fields emit warnings if empty.

Validate at any time without promoting:
```bash
rh-skills validate diabetes-screening l2/concept-1.yaml
```

### Step 4: Promote to L3 (L2 → L3)

When your L2 artifacts are ready, combine them into a computable L3 artifact:

```bash
rh-skills promote diabetes-screening \
  --combine screening-criteria risk-factors \
  --output screening-guideline
```

This scaffolds `l3/screening-guideline.yaml` pre-populated from the contributing L2 artifacts, then validates it against the L3 schema.

Fill in the L3 content — pathways, actions, value sets — following the schema in `contracts/artifact-schemas.md`.

### Step 5: Write the skill prompt

Open `skills/diabetes-screening/SKILL.md` and write the agent-facing prompt that uses the L3 artifact content. This is the instruction an AI agent will receive when invoking the skill.

```markdown
---
name: "diabetes-screening"
description: "Evidence-based diabetes screening criteria and recommendations"
compatibility: "Requires L3 artifact at skills/diabetes-screening/l3/"
metadata:
  author: "Jane Smith"
  domain: "Diabetes Management"
---

## User Input

Patient context: $ARGUMENTS

## Instructions

Using the screening criteria defined in the L3 artifact, evaluate whether
the described patient meets criteria for diabetes screening and provide
a structured recommendation.

Reference: skills/diabetes-screening/l3/screening-guideline.yaml
```

### Step 6: Add test fixtures

```bash
mkdir -p skills/diabetes-screening/fixtures/basic-case
```

Create `fixtures/basic-case/input.yaml`:
```yaml
description: "Standard adult patient with risk factors"
messages:
  - role: user
    content: |
      45-year-old with BMI 28, family history of Type 2 diabetes,
      no prior screening. Should we screen?
```

Create `fixtures/basic-case/expected.yaml`:
```yaml
comparison_mode: keywords
expected_output: "A1C fasting glucose screening recommended"
tags:
  - basic
  - adult
```

### Step 7: Run local tests

```bash
rh-skills test diabetes-screening
```

Output:
```
Running tests for: diabetes-screening
Provider: ollama (mistral)

  ✓ basic-case    PASSED  (keywords, 1.4s)

Results: 1 passed, 0 failed, 0 errored
```

## LLM Provider Configuration

| Provider | Environment Variables |
|----------|----------------------|
| Ollama (local, default) | `LLM_PROVIDER=ollama`, `OLLAMA_ENDPOINT`, `OLLAMA_MODEL` |
| Anthropic | `LLM_PROVIDER=anthropic`, `ANTHROPIC_API_KEY`, `ANTHROPIC_MODEL` |
| OpenAI-compatible | `LLM_PROVIDER=openai`, `OPENAI_API_KEY`, `OPENAI_ENDPOINT`, `OPENAI_MODEL` |

Recommended `.env` setup:
```bash
# .env (git-ignored)
LLM_PROVIDER=ollama
OLLAMA_ENDPOINT=http://localhost:11434
OLLAMA_MODEL=mistral
```
