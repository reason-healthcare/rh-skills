## Entity 1: Source Entry (`discovery-plan.yaml` â€” sources[])

```yaml
sources:
  - name: "string"           # kebab-case identifier; used as filename stem in sources/
    title: "string"          # human-readable title of the source
    type: "string"           # see Source Type Taxonomy (spec FR-003)
    url: "string"            # canonical URL (required; may be abstract page for auth sources)
    access: "string"         # open | authenticated | manual
    evidence_level: "string" # ia | ib | iia | iib | iii | iv | v | expert-consensus
    open_access: false       # true if available without auth (PMC, preprint, government site)
    recommended: false       # true for high-value authenticated sources worth manual retrieval
    auth_note: "string"      # required if access: authenticated; describes institution/method
    rationale: "string"      # 1â€“2 sentences: why this source is relevant to the topic
    pmid: "string"           # optional; PubMed ID if source found via PubMed
    pmcid: "string"          # optional; PMC ID if full text available in PMC
    nct_id: "string"         # optional; ClinicalTrials.gov NCT ID if applicable
    downloaded: false        # true when hi ingest implement --url has run for this source
    sha256: "string"         # populated by hi ingest implement --url
    local_file: "string"     # populated by hi ingest implement --url; relative path in sources/
```

**Validation rules**:
- `name`: required; kebab-case; unique within plan
- `type`: must be one of the Source Type Taxonomy values (spec FR-003)
- `access`: required; one of `open | authenticated | manual`
- `auth_note`: required when `access: authenticated`; error in verify mode if missing
- `open_access: true` requires `access: open`
- `evidence_level`: required; must be one of the Evidence Level Taxonomy values
- Plan must have â‰Ą5 sources (warn below), â‰¤25 sources (surplus goes to expansion suggestions)

## Entity 2: Discovery Plan (`topics/<name>/process/plans/discovery-plan.yaml`)

Pure YAML file â€” no frontmatter delimiters (`---`). This is the machine-readable source of truth consumed by `hi validate --plan` and `hi-ingest`. Human-editable between sessions.

```yaml
topic: "string"              # Matches topics/<name> directory
version: "string"            # Semantic version; starts at "1.0"
created: "ISO-8601 date"     # When plan was first saved
last_updated: "ISO-8601 date" # When plan was last modified in this session
status: "string"             # draft | approved | superseded
session_id: "string"         # Unique identifier for the discovery session
domain_advice:
  coverage_gaps: []          # List of coverage areas not yet addressed
  expansion_suggestions: []  # 3â€“7 strings: areas the user may want to research next
sources: []                  # Source entries (see Entity 1)
research_expansion:
  - area: "string"           # Area name (e.g., "pediatric subgroup analysis")
    rationale: "string"      # Why relevant; not auto-added to sources
```

**State transitions**:
- `draft` â†’ `approved`: user explicitly approves save
- `approved` â†’ `superseded`: new session replaces existing plan (old file archived with date suffix)
